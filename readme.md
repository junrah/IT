Java Mini Project
=

### 자바 소켓통신과 GUI를 활용한 빙고 게임

- [Javadoc](https://junrah.github.io/IT/miniProject/doc/index.html)  

#### 개발환경
- JDK : Java 1.8  
- IDE : 이클립스  

#### 잘한 점
빙고가 발생할 수 있는 가능성을 모두 확인하여 빙고의 칸 개수가 늘어나도 정상적으로 작동할 수 있도록 설계했습니다.  
빙고 칸마다 플레이어가 원하는 단어를 작성하고 원하는 곳에 배치할 수 있도록 설계하였습니다.  
게임 승패 결정 후 서버와 각 플레이어간의 신호를 주고받음으로써 정상적으로 종료될 수 있도록 설계하였습니다.  

#### 힘들었던 점 
전체적인 설계 구조를 구상하지않은 채 생각나는대로 구현하다보니 정작 중요하다 생각하는 기능을 넣을때마다 오류가 발생하였습니다.  
따라서 설계를 세분화하여 중요하다 생각하는 핵심 기능을 먼저 구현하고 뼈대에 살을 붙이는 방식으로 풀어나가야 했습니다.  

나의 공격이 나의 승리로 이어지기도 하지만 상대의 공격으로 내 빙고가 완성되는 두 가지 모두 설계하는것이 쉽지않았습니다.  
또한 두 개의 콘솔로 이 두가지 경우가 각 플레이어에게서 일어날 경우를 확인하는데 시간이 많이 소요되었습니다.  
빙고 완성시 플레이어가 서버로 빙고 완성 신호를 보내면 서버측에서 승리와 패배의 신호를 각 플레이어에게 보내고  
각 플레이어는 승리 및 패배 신호에 맞게 게임을 종료하는 방법으로 구현했습니다.  

#### 보완해야할 점 
서버와 클라이언트 각 하나의 스레드를 활용한 코드이기때문에 서버에서 준비완료 신호를 순서대로 받아들입니다.  
따라서 플레이어의 준비신호가 모두 서버로 전달되면 게임을 시작하는 방식으로 구현했지만  
플레이어마다 준비완료하는 시점이 다를 수 있기 때문에 서버에서 스레드를 활용하여  
랜덤하게 들어오는 준비완료신호를 처리하는 방법도 좋을 것 같습니다.  

버튼에 해당하는 텍스트만 전달하는 것이아니라 [x],[y]좌표도 함께 전달하는 방식으로  
클라이언트 화면에서 상대의 진행상황을 구현하는게 좋을 것 같습니다.
